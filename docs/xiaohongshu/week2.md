# 小红书 Week 2：场景切换让路

**标题**：我用 4 周做一个 AI 面试 Agent #2：让 agent 像真面试官那样给台阶

**正文**：

第 2 周搞 U-loop 多轮追问。最骄傲的不是追问深度本身，而是「场景切换」UX。

真实场景：面试官问「举一个实习中体现领导力的例子」，你卡壳半天答了句「我帮组里同事改 PPT」——典型例子撑不住考察意图。

通用 ChatGPT 怎么做？两种结局：(1) 死磕这个例子追问到尴尬；(2) 直接放弃下一题。

我的 agent 会主动给台阶：「这个例子可能不够典型，要不你说说项目 / 校园 / 课外里类似的事？」

机制：释放场景维度（实习/项目/校园/生活）保留考察意图（领导力/决策/数据驱动）。如果连考察意图本身都没料，再松绑到相邻意图（"主动推动事情发生" 也算同维度证据）。

每题最多切 2 次场景，避免无限漂移。

下周做前端，让这套对话流跑在浏览器里。
