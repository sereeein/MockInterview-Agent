# 小红书 Week 4：评估 + 上线 + 简历金句

**标题**：我用 4 周做一个 AI 面试 Agent #4：vs 裸 Claude 盲评胜率 <X>%

**正文**：

第 4 周收尾。

**评估**：搭了 LLM-as-judge pipeline。每个简历 × JD 对：
1. 出题相关性（每道题 0-3 分）
2. 追问命中率（用 LLM 模拟"中等质量"用户答 → judge 判 agent 追问是否击中最弱维度）
3. vs 裸 Claude 盲评胜率（同输入两份输出，judge 不知道哪个是我的）

跑了 8 个 pair × ~35 LLM 调用 = ~280 calls。结果：
- 相关性 **<X>** / 3（目标 ≥ 2.2）
- 追问命中率 **<X>%**（目标 ≥ 70%）
- vs 裸 Claude 盲评胜率 **<X>%**（目标 ≥ 70%）

**部署**：前端 Vercel + 后端 Railway，prompt caching 让真实成本可接受。

**给同样想做求职项目的朋友 3 条建议**：
1. 先想垂直化策略（只服务 PM/数据/AI 比通用面试工具更值钱）
2. 一定要做 eval pipeline（量化 prompt 调优 ≥ "感觉对了"）
3. 简历金句反推工程取舍（让"vs baseline 胜率"成为决策灯塔）
